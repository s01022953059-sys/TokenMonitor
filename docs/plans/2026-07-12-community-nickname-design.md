# 社区昵称设计

## 目标

允许用户在社区排行页修改公开昵称，同时保持匿名身份、设备凭据、历史用量和排名连续。昵称是可修改的展示属性，`User_XXXXXXXX` 是不可修改的底层身份。

## 交互

- 社区页身份区域显示公开昵称；未设置时回退显示匿名 ID。
- 昵称下方始终保留较弱的 `ID: User_XXXXXXXX`，避免展示名与身份混淆。
- 使用铅笔图标进入原地编辑，不增加新弹窗。
- 编辑态包含输入框、字数、确认和取消图标；保存期间禁用重复提交。
- 客户端即时提示基础格式错误，VPS 返回重名、风险名称、冷却期和服务错误。
- 保存失败保留原昵称与输入；成功后刷新身份区域、排行榜和“你”标记。

## 名称规则

- 允许中文、ASCII 英文字母、数字和下划线，长度 2–16 个 Unicode 字符。
- 至少包含一个中文或英文字母；禁止纯数字、纯下划线。
- 输入先去除首尾空白并执行 Unicode NFKC；唯一键再转小写。
- 禁止不可见字符、零宽字符、双向控制符、换行、Emoji 和其他符号。
- 禁止连续四个以上相同字符。
- 禁止 `User_` 前缀及官方、管理员、客服、系统、支持、版主等身份冒充名称。
- 保护名称使用忽略大小写、下划线、全角差异及常见 `0/O`、`1/I` 替换的骨架检查。
- 拦截明显违规词、网址、手机号、QQ、微信及“加我/联系我”等广告导流组合。
- 风险词由 VPS 配置文件维护，客户端不包含完整词表，服务端是最终裁决方。

## 唯一性与频率

- `canonical_name` 在全社区全局唯一，SQLite 唯一索引负责并发裁决。
- 首次设置不限；设置后每 7 天只能修改一次，时间以 VPS UTC 为准。
- 改名后的旧名称保护 30 天，期间任何其他身份都不能占用。
- 当前昵称长期保留，不因当天未同步或长期不活跃而自动释放。

## VPS 数据模型

SQLite 路径：`/var/lib/token-monitor-community/community.db`。

`profiles`：

- `user_id`：主键，对应不可变匿名 ID。
- `display_name`：公开昵称。
- `canonical_name`：规范化唯一键，唯一索引。
- `created_at`：首次设置时间。
- `changed_at`：最近修改时间。

`name_reservations`：

- `canonical_name`：旧名称唯一键。
- `owner_id`：原身份。
- `reserved_until`：30 天保护截止时间。

## 接口与鉴权

新增 `POST /v1/profile`：

```json
{
  "id": "User_ABCDEFGH",
  "device_secret": "...",
  "display_name": "鹏帅"
}
```

中继从 GitCode 当前报告读取 `auth_hash`，使用常量时间比较验证设备凭据。未完成首次社区同步的身份不能改名。

更新在 SQLite `BEGIN IMMEDIATE` 事务内完成：校验格式与风险词、检查当前昵称、7 天冷却、全局唯一和旧名保护，随后写入 GitCode 报告；GitCode 写入失败则回滚事务，不占用新昵称。成功后提交 SQLite，并返回规范化后的展示名与下次可修改时间。

错误码保持稳定：`credential_invalid`、`profile_not_found`、`name_invalid`、`name_reserved`、`name_taken`、`rename_cooldown`、`storage_unavailable`、`upload_failed`。

macOS/Python 与 Windows/Go 本地服务新增 `POST /api/community/profile`，只负责读取本机设备凭据、转发昵称并清除社区缓存。浏览器不接触设备密钥。

## 安全与隐私

- 昵称是公开数据，页面编辑态明确提示“昵称将公开显示”。
- 前端始终通过 `textContent` 或现有 `escapeHtml` 输出昵称，禁止 HTML 注入。
- 中继限制请求体大小、Content-Type、字段集合和超时；Nginx 与服务端共同限流。
- 设备密钥不写入日志、SQLite、GitCode 或 API 响应；只保存现有 SHA-256 哈希。
- 重名响应不泄露设备或报告信息；风险词统一返回通用原因。
- 不提供修改底层匿名 ID 的接口，避免借改名绕过身份鉴权和重复统计。

## 验证门禁

- Python、Windows Go 和中继共用等价规则测试：中英文、边界长度、NFKC、大小写、全角、纯数字、不可见字符、双向字符、Emoji、XSS、重复字符、保护名称、广告联系方式。
- SQLite 测试覆盖并发抢名、7 天冷却、旧名 30 天保护、同名无变化和 GitCode 失败回滚。
- 鉴权测试覆盖错误凭据、未知身份、旧版无凭据报告。
- 端到端测试使用两个临时身份验证抢名冲突、改名传播、排行榜显示和清理。
- 浏览器验证桌面与 390px 窄屏的编辑、取消、保存、错误提示、键盘操作和无横向溢出。
- 只有全部门禁通过且鹏帅明确要求发版，才提升版本并发布 macOS/Windows 安装包。
